# AAPE.JP 全站商品爬蟲(草稿)

## 這是什麼

一支會自動跑遍 aape.jp 所有分類、抓出每件商品的名稱/價格/顏色/尺寸的爬蟲,
輸出的 `aape_all_products.json` 可以直接貼進 `daigou-shop.html` 後台的
「批次匯入商品」功能。

## ⚠️ 這是草稿,還沒實際連網測試過

程式裡標了好幾處 `TODO: 確認實際 selector` 的地方 —— 這些是我先合理猜測
網站的 HTML 結構寫的,**一定要對照真實網頁調整過才能正確運作**。

## 建議交給 Claude Code 做的事

1. **安裝套件**:`pip install -r requirements.txt`
2. **小範圍測試**:先用 `python scrape_aape.py --limit 5 --skip-detail` 只抓 5 件、
   不查詳細頁,確認分類、列表抓取的邏輯有沒有跑起來
3. **對照真實 HTML 調整 selector**:如果抓不到東西或抓錯,請 Claude Code
   打開瀏覽器開發者工具(F12)或直接看網頁原始碼,把程式裡的
   `soup.select(...)` 換成真正對應的 class 名稱
4. **確認商品詳細頁的顏色/尺寸/庫存怎麼標示**:這是最不確定的部分,
   不同網站寫法差異很大,需要實際打開一兩個商品頁比對
5. **調整完後正式執行**:`python scrape_aape.py`,跑完會產生
   `aape_all_products.json`

## 執行時間提醒

- 商品可能有好幾百件,每件詳細頁都要開一次,加上禮貌性的延遲(0.8 秒/次請求),
  跑完全部可能要一段時間,不是幾秒鐘的事,請有耐心
- 想先看看效果,可以用 `--limit 20` 只抓一小部分測試

## 使用前提醒

- 大型品牌官網通常會在服務條款裡對自動化抓取有一些規範,正式大量使用前
  建議留意一下該網站的條款
- 重量是用商品名稱關鍵字概略估的(已加上安全緩衝),不是官方數據,
  建議正式上架前抽件確認,尤其是差異較大的品項(包款、外套)
- 庫存資訊：程式目前只能判斷「有貨/售完」,抓不到精確的庫存件數
  (官網通常也不會公開顯示精確庫存數字)
- 抓下來的是「當下」的快照,不會自動更新,商品下架、補貨、調價都不會
  反映在這份 JSON 裡,除非重新執行一次
